1. Define primitive and non-primitive data types with example.
  1. Primitive Data Types: Primitive data types are basic data types built into the programming language. They are directly supported by the language and are used to define variables that store simple values. Examples include :
    • Integer: Used to store whole numbers. Example: int x = 10;
    • Floating-point: Used to store decimal numbers. Example: float pi = 3.14;
    • Character: Used to store single characters. Example: char ch = 'A';
    • Boolean: Used to store true or false values. Example: bool isTrue = true;
  2. Non-Primitive (Reference) Data Types: Non-primitive data types, also known as reference data types, are types that are not defined by the programming language but are created by the programmer. They are typically used to store collections of data or complex objects. Non-primitive data types store references (memory addresses) to the location of the actual data in memory. Examples of non-primitive data types include:
    • Array: A collection of elements of the same type. Example: int arr[5] = {1, 2, 3, 4, 5};
    • String: A sequence of characters. Example: string name = "John"; (in languages like C++ or Java)
    • Struct: A user-defined data type that groups together variables of different types. struct Person { char name[50]; int age; };
    • Class/Object: In object-oriented programming languages like Java or C++, classes are used to define blueprints for objects.
      class Car { String color; int year; }

These are just a few examples of primitive and non-primitive data types. Different programming languages may have additional data types or may use different terminology, but the fundamental concepts remain similar across languages.

3. Write recursive algorithm to compute factorial of a given number. Which data structure can be used to implement this algorithm?
#include <iostream>
using namespace std;

// Recursive function to compute factorial
int factorial(int n) {
    // Base case: if n is 0 or 1, factorial is 1
    if (n == 0 || n == 1) {
        return 1;
    }
    // Recursive case: multiply n by factorial of (n-1)
    else {
        return n * factorial(n - 1);
    }
}
int main() {
    int num;
    cout << "Enter a number: ";
    cin >> num;
    cout << "Factorial of " << num << " is: " << factorial(num) << endl;
    return 0;
}

The function calls are managed using the call stack, which is a system-level data structure maintained by the compiler/runtime environment. Each function call is added to the call stack, and when a function returns, it is removed from the stack. So, the call stack effectively manages the recursion in this case.

5. Describe: (1) Recursion (2) Priority Queue (3) Tower of Hanoi
  1. Recursion: Recursion is a programming technique where a function calls itself in order to solve a problem. It breaks down a problem into smaller sub-problems of the same type, solving each one recursively until reaching a base case, which is a simple case that can be solved directly without further recursion. Recursion is commonly used in algorithms such as factorial calculation, Fibonacci sequence generation, and tree traversal.
  2. Priority Queue: A priority queue is an abstract data type similar to a regular queue, but with each element assigned a priority. Elements with higher priority are dequeued before those with lower priority, regardless of their order of arrival. Priority queues are often implemented using heaps, where the highest priority element is at the root of the heap. Priority queues are commonly used in various applications such as scheduling algorithms, graph algorithms like Dijkstra’s algorithm, and Huffman coding.

A queue is a linear data structure that follows the First In, First Out (FIFO) principle, meaning that the element added first will be the one to be removed first. It’s similar to a queue of people waiting in line, where the person who joined the queue first will be the first to be served.

Drawback of a Simple Queue :
One drawback of a simple queue (implemented using an array or linked list) is that once an element is dequeued (removed), the space it occupied is wasted and cannot be reused. This can lead to inefficient use of memory, especially in scenarios where elements are frequently enqueued and dequeued, causing the queue to grow and shrink dynamically.

18. How does priority queue work?

A priority queue is an abstract data type that behaves similarly to the normal queue except that each element has some priority, i.e., the element with the highest priority would come first in a priority queue. The priority of the elements in a priority queue will determine the order in which elements are removed from the priority queue.
The priority queue supports only comparable elements, which means that the elements are either arranged in an ascending or descending order.
For example, suppose we have some values like 1, 3, 4, 8, 14, 22 inserted in a priority queue with an ordering imposed on the values is from least to the greatest. Therefore, the 1 number would be having the highest priority while 22 will be having the lowest priority.

19. Describe the advantages of linked list over array
  1. Dynamic Size: One of the most significant advantages of linked list over arrays is that linked lists can grow or shrink dynamically during runtime. This means that the size of a linked list can be adjusted to accommodate new elements or remove existing elements without having to allocate or deallocate a fixed-size block of memory, as is the case with arrays.
  2. Efficient Insertion and Deletion: Linked lists allow efficient insertion and deletion of elements at any position in the list, whereas arrays require shifting of elements when a new element is added or removed, which can be slow and inefficient for large arrays.
  3. Memory Efficiency: Linked lists use memory more efficiently than arrays. In an array, all elements must be stored in contiguous memory locations, even if some of the elements are not used. In contrast, linked lists only allocate memory for the elements that are used, which can save memory in cases where the size of the data set is unknown or varies over time.
  4. Easy Implementation of Abstract Data Types: Linked lists are easy to use and implement when implementing abstract data types such as stacks, queues, and trees. These data structures require frequent insertion and deletion of elements, which is a task in which linked lists are well-suited.
  5. More Efficient Sorting: In some cases, linked lists can be more efficient for sorting algorithms than arrays. This is because linked lists do not require swapping elements like arrays, which can be time-consuming for large arrays.

In short, there are several advantages of linked list over arrays, such as dynamic size, efficient insertion and deletion, memory efficiency, easy implementation of abstract data types, and more efficient sorting in some cases.

33. List the advantages of a doubly linked list over singly linked list.
  1. Easy Forward and Backward Movement: Doubly linked lists allow you to move both forward and backward through the list without needing to jump through hoops. Each item has a pointer to the next and previous items, making navigation a breeze.
  2. Quick Node Removal: Removing a node from a doubly linked list is faster compared to singly linked lists. You don’t need to search for the previous node; you already have it stored.
  3. Convenient Insertions and Deletions: You can easily add or remove items from both the front and back of the list. This flexibility comes in handy when you need to frequently modify the list’s beginning or end.
  4. Flexible Manipulation: With two pointers in each node, doubly linked lists offer more flexibility in manipulating the list’s structure. Tasks like merging lists or rearranging elements become simpler.
  5. Handy for Algorithms: In some algorithms, you need to backtrack to previous nodes. Doubly linked lists make this easy because you can quickly access the previous nodes.
